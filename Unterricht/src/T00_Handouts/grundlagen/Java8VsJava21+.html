Java 8 vs Java 21: Neue Features über die Jahre

Java hat sich seit Version 8 deutlich weiterentwickelt. Während Java 8 als “Meilenstein” für funktionale Programmierung gilt, bietet Java 21 zahlreiche moderne Features, die Sprache und Plattform deutlich leistungsfähiger machen.

Wichtige Neuerungen in Java 8:

  • Lambda-Ausdrücke (Funktionen als Werte, Kurzschreibweise für anonyme Methoden)
// Lambda für eine Funktion, die eine Zahl verdoppelt
Function<Integer, Integer> verdoppeln = x -> x * 2;
System.out.println(verdoppeln.apply(5)); // Ausgabe: 10
  • Streams API (Effiziente Verarbeitung von Collections mittels Pipeline-Operationen wie filter, map, reduce)
List<String> namen = List.of("Anna", "Ben", "Clara", "David");
List<String> gefiltert = namen.stream()
                              .filter(n -> n.startsWith("C"))
                              .map(String::toUpperCase)
                              .toList();
System.out.println(gefiltert); // Ausgabe: [CLARA]
  • Functional Interfaces (Runnable, Comparator etc. – Schnittstellen mit genau einer abstrakten Methode)
Comparator<String> compareByLength = (s1, s2) -> s1.length() - s2.length();
List<String> liste = List.of("Banane", "Apfel", "Kirsche");
liste.sort(compareByLength);
System.out.println(liste); // Ausgabe: [Apfel, Banane, Kirsche]
  • Default Methods (Standard-Implementierungen in Interfaces)
interface Fahrzeug {
    default void starten() {
        System.out.println("Fahrzeug startet");
    }
}

class Auto implements Fahrzeug {}
new Auto().starten(); // Ausgabe: Fahrzeug startet
  • Neue Date & Time API (java.time für bessere Zeit- und Datumsverarbeitung)
LocalDate heute = LocalDate.now();
LocalDate geburtstag = LocalDate.of(1990, Month.JULY, 15);
Period alter = Period.between(geburtstag, heute);
System.out.println("Alter: " + alter.getYears());

Folgendes wird im Unterricht nicht genauer behandelt, kann aber für den ein oder anderen sehr interessant sein:

✅ Optional (Vermeidung von null, bessere Kontrolle über Fehlende Werte)

Javabeginners - Optionals

Motivation

  • In Java gibt es oft Werte, die nicht vorhanden sein könnten (null).

  • Direkter Zugriff auf null führt zu NullPointerExceptions.

  • Optional hilft, explizit mit fehlenden Werten umzugehen und den Code sicherer zu machen.

Was ist Optional?

  • Ein Container, der entweder einen Wert enthält oder leer ist.

  • Vermeidet direkte null-Checks und macht Absichten klar.

Vorteile

  • Reduziert NullPointerExceptions

  • Klarere Code-Struktur

  • Ausdrucksstarke Methoden wie map(), ifPresent(), orElse()

import java.util.Optional;

public class OptionalExample {
    public static void main(String[] args) {
        Optional<String> name = Optional.ofNullable(getName());

        // Traditionell
        if (name.isPresent()) {
            System.out.println("Name: " + name.get());
        }

        // Moderne Variante
        name.ifPresent(n -> System.out.println("Name: " + n));

        // Standardwert, falls leer
        String defaultName = name.orElse("Unbekannt");
        System.out.println("Name: " + defaultName);
    }

    static String getName() {
        return null; // simuliert einen fehlenden Wert
    }
}

✅ Parallel Streams und Fork/Join Framework (Einfachere parallele Programmierung)

Motivation

  • Streams ermöglichen funktionales Arbeiten mit Collections.

  • Bei großen Datenmengen kann die Verarbeitung zeitaufwendig sein.

  • Parallel Streams teilen die Arbeit automatisch auf mehrere Threads auf (was nicht heißt, dass es automatisch schneller ist)

Vorteile

  • Einfache Parallelisierung ohne explizite Threads

  • Leistungssteigerung bei CPU-intensiven Operationen

  • Integration in bestehende Stream-APIs

import java.util.List;
import java.util.stream.IntStream;

public class ParallelStreamExample {
    public static void main(String[] args) {
        // Große Datenmenge
        List<Integer> zahlen = IntStream.rangeClosed(1, 10_000_000).boxed().toList();

        // Serieller Stream
        long start = System.currentTimeMillis();
        long sum = zahlen.stream()
                         .mapToLong(n -> n * n)
                         .sum();
        long end = System.currentTimeMillis();
        System.out.println("Summe der Quadrate (seriell): " + sum);
        System.out.println("Dauer seriell: " + (end - start) + " ms");

        // Paralleler Stream
        start = System.currentTimeMillis();
        long parallelSum = zahlen.parallelStream()
                                 .mapToLong(n -> n * n)
                                 .sum();
        end = System.currentTimeMillis();
        System.out.println("Summe der Quadrate (parallel): " + parallelSum);
        System.out.println("Dauer parallel: " + (end - start) + " ms");
    }
}

Java 21: Die moderne Entwicklerplattform

Wesentliche Neuerungen von Java 9–21:

✅ Modulsystem (Java 9): Bessere Strukturierung großer Projekte

Theorie:

  • Java-Projekte können in Module aufgeteilt werden.

  • Jedes Modul definiert, welche Pakete es exportiert und welche von außen sichtbar sind.

  • Hilft bei klarer Struktur, Wiederverwendbarkeit und Kapselung.

✅Local Variable Type Inference – var (Java 10): Einfachere Schreibweise für Variablen, Typ wird vom Compiler abgeleitet

Theorie:

  • Compiler leitet den Typ automatisch ab, kein explizites Schreiben nötig.

  • Spart Tippaufwand, macht Code kompakter.

  • Funktioniert nur lokal innerhalb von Methoden, nicht für Felder oder Parameter.

public static void main(String[] args) {
        var name = "Max";  // Compiler weiß: String
        var zahl = 42;     // Compiler weiß: int
        System.out.println(name + " " + zahl);
}

✅ Text Blocks (Java 13): Mehrzeilige Zeichenketten ohne Escape-Orgien

  • Erlaubt mehrzeilige Strings, ohne \n oder viele Anführungszeichen.

  • Gut für SQL, JSON oder HTML.

String html = """
    <html>
        <body>
            <h1>Hello, World!</h1>
        </body>
    </html>
    """;
System.out.println(html);

✅ Records (Java 14/16): Immutable Datenträger-Klassen mit minimalem Code

Theorie

  • Records sind spezielle Klassen, die primär Daten transportieren (Data-Carrier).

  • Sie sind immutable (unveränderlich) – ihre Felder können nach Erstellung nicht mehr geändert werden.

  • Automatisch generiert der Compiler:

    • Konstruktor

    • Getter-Methoden (name(), age() usw.)

    • equals() und hashCode()

    • toString()

Vorteile:

  • Weniger Boilerplate-Code als bei normalen Klassen.

  • Klarer Fokus auf die Daten, nicht auf Logik.

  • Ideal für DTOs, Value Objects, einfache Konfigurationen.

Einschränkungen:

  • Felder sind automatisch final.

  • Records können nicht von anderen Klassen erben (nur von java.lang.Record).

  • Sie können jedoch Interfaces implementieren.

Zuvor: 

// Klassische Java-Klasse für Daten
public class Person {
    private final String name;
    private final int alter;

    // Konstruktor
    public Person(String name, int alter) {
        this.name = name;
        this.alter = alter;
    }

    // Getter
    public String getName() {
        return name;
    }

    public int getAlter() {
        return alter;
    }

    // equals & hashCode
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Person)) return false;
        Person person = (Person) o;
        return alter == person.alter && name.equals(person.name);
    }

    @Override
    public int hashCode() {
        return Objects.hash(name, alter);
    }

    // toString
    @Override
    public String toString() {
        return "Person[name=" + name + ", alter=" + alter + "]";
    }
}

// Nutzung
public class Main {
    public static void main(String[] args) {
        Person p = new Person("Anna", 25);
        System.out.println(p);
    }
}

Ab Java 14+

// Definition eines Records
record Person(String name, int alter) {}

// Nutzung
public class Main {
    public static void main(String[] args) {
        Person p = new Person("Anna", 25);

        // Getter automatisch generiert
        System.out.println("Name: " + p.name());
        System.out.println("Alter: " + p.alter());

        // toString automatisch generiert
        System.out.println(p); // Ausgabe: Person[name=Anna, alter=25]

        // equals/hashCode automatisch generiert
        Person p2 = new Person("Anna", 25);
        System.out.println(p.equals(p2)); // true

        // Immutable: folgende Zeile würde Fehler erzeugen
        // p.name = "Ben"; // nicht erlaubt
    }
}

Man kann aber auch zusätzliche Methoden oder einen kompakten Konstruktor hinzufügen: 

record Person(String name, int alter) {
    // Kompakter Konstruktor
    public Person {
        if (alter < 0) throw new IllegalArgumentException("Alter muss positiv sein");
    }

    // Zusätzliche Methode
    public String begruessung() {
        return "Hallo, mein Name ist " + name;
    }
}

public class Main {
    public static void main(String[] args) {
        Person p = new Person("Max", 30);
        System.out.println(p.begruessung());
    }
}

✅ Pattern Matching (Java 16/17/21): Vereinfachte Bedingungen bei Typprüfungen (instanceof, switch)

  • In klassischen Java-Versionen muss man oft zwei Schritte machen:

    1. Prüfen, ob ein Objekt vom richtigen Typ ist (instanceof)

    2. Dann explizit casten

  • Problem: Boilerplate-Code, weniger lesbar

  • Pattern Matching kombiniert Prüfung und Zuweisung in einer Zeile

  • Funktioniert sowohl mit instanceof als auch mit switch (Java 17+)

Zuvor: 

Object obj = "Hallo, Welt!";
if (obj instanceof String) {
    String s = (String) obj; // cast nötig
    System.out.println(s.toUpperCase());
}

Ab Java 16: 

Object obj = "Hallo, Welt!";

if (obj instanceof String s) {  // Prüfung + Zuweisung in einer Zeile
    System.out.println(s.toUpperCase());
}

Zuvor:

String result;
switch (day) {
    case "MONDAY":
        result = "Start";
        break;
    default:
        result = "Other";
}

Ab Java 21:

String result = switch (day) {
    case "MONDAY" -> "Start";
    default -> "Other";
};
Object obj = 123;

switch (obj) {
    case String s -> System.out.println("String: " + s);
    case Integer i -> System.out.println("Integer: " + i);
    default -> System.out.println("Andere Art: " + obj);
}

✅Sealed Classes (Java 15/17): Kontrolle, welche Klassen von einer Basisklasse erben dürfen

Theorie

  • Normalerweise kann jede Klasse eine andere Klasse erweitern.

  • Sealed Classes erlauben, die erlaubten Subtypen explizit festzulegen.

  • Vorteile:

    • Bessere Kontrolle über Vererbungshierarchie

    • Erleichtert Pattern Matching und Switches

    • Sicherheit + Wartbarkeit

sealed class Shape permits Circle, Rectangle {}  // nur diese beiden dürfen erben

final class Circle extends Shape {}
final class Rectangle extends Shape {}

✅Virtual Threads (Java 19/21): Nebenläufigkeit mit Millionen Threads, leichtgewichtig und performant

Theorie

  • Klassische Threads sind schwergewichtig (hoher Speicherverbrauch, Thread-Limit)

  • Virtual Threads sind extrem leicht und können Millionen gleichzeitig laufen

  • Ziel: leichte, skalierbare Nebenläufigkeit, ideal für Server, Parallelverarbeitung, IO-intensive

Runnable task = () -> System.out.println("Thread: " + Thread.currentThread());

Thread vThread = Thread.ofVirtual().start(task); // leichter Thread
vThread.join(); // Warten auf Fertigstellung
import java.nio.file.*;
import java.util.List;

public class VirtualThreadsFileExample {
    public static void main(String[] args) throws Exception {
        List<Path> files = List.of(
            Path.of("file1.txt"),
            Path.of("file2.txt"),
            Path.of("file3.txt")
        );

        List<Thread> threads = files.stream()
            .map(file -> Thread.ofVirtual().start(() -> readFile(file)))
            .toList();

        for (Thread t : threads) t.join();
    }

    static void readFile(Path path) {
        try {
            String content = Files.readString(path);
            System.out.println("Datei " + path + " hat " + content.length() + " Zeichen gelesen von " + Thread.currentThread());
        } catch (Exception e) {
            System.out.println("Fehler beim Lesen von " + path);
        }
    }
}

✅String Templates (nur! Java 21): Komfortablere String-Interpolation

Theorie

  • Bisher: + oder String.format() für Variablen in Strings

  • Jetzt: direkte Interpolation mit STR....

  • Vorteil: lesbarer, kürzer, weniger Fehleranfällig

String name = "Max";
int alter = 30;
String text = STR."Name: \{name}, Alter: \{alter}";
System.out.println(text); // Name: Max, Alter: 30

✅ Sequenced Collections (Java 21): Zugriff auf erstes/letztes Element, bessere Listen-APIs

Sequenced Collections in Java | Baeldung

Theorie

  • Standard-Collections wie HashMap haben keine garantierte Einfüge-Reihenfolge.

  • Sequenced Collections garantieren, dass Elemente in der Reihenfolge des Einfügens gespeichert und iteriert werden.

  • Erleichtert Zugriff auf erstes/letztes Element und deterministische Iterationen.

  • Unterstützt z. B. SequencedMap und SequencedSet.

Vorteile:

  • Vorher: Man musste für erstes/letztes Element oft LinkedList oder Iteratoren benutzen.

  • Jetzt: Direkter Zugriff über API (first(), last()), einfacher Code.

import java.util.*;

public class SequencedExample {
    public static void main(String[] args) {

        List<String> namen = List.of("Anna", "Ben", "Clara");
        System.out.println("Erstes: " + namen.get(0));
        System.out.println("Letztes: " + namen.get(namen.size() - 1));

        // Sequenced Map mit Einfüge-Reihenfolge
        Map<String, Integer> map = new LinkedHashMap<>(); // SequencedMap Interface in Java 21
        map.put("Alice", 30);
        map.put("Bob", 25);
        map.put("Clara", 28);

        // Zugriff auf erstes Element
        String firstKey = map.keySet().iterator().next();
        System.out.println("Erstes Element: " + firstKey + " -> " + map.get(firstKey));

        // Zugriff auf letztes Element
        String lastKey = null;
        for (String key : map.keySet()) lastKey = key;
        System.out.println("Letztes Element: " + lastKey + " -> " + map.get(lastKey));

        // Iteration in Einfüge-Reihenfolge
        System.out.println("Alle Elemente in Einfüge-Reihenfolge:");
        map.forEach((k,v) -> System.out.println(k + " -> " + v));
    }
}

✅Unnamed Variables/Patterns (Java 21, Preview): _ als Platzhalter, wo Ergebnisse verworfen werden

Theorie

  • _ dient als Platzhalter, wenn man einen Wert ignorieren möchte.

  • Praktisch bei Destructuring von Records, Pattern Matching oder Switches.

  • Vorteil: Keine unnötigen Variablen, klarer Code.

// Record für einen Punkt im 2D-Raum
record Point(int x, int y) {}

public class Main {
    public static void main(String[] args) {
        Point p = new Point(5, 10);

        // Wir brauchen nur y, x ignorieren wir
        var Point(_, y) = p;
        System.out.println("Nur y interessiert: " + y);  // Ausgabe: 10
    }
}

Object obj = new Point(3, 7);

switch (obj) {
    case Point(_, int yVal) -> System.out.println("y ist " + yVal);
    default -> System.out.println("Kein Point");
}


// Record für Koordinaten
record Point(int x, int y, int z) {}

public class Main {
    public static void main(String[] args) {
        Point p = new Point(5, 10, 15);

        // Wir interessieren uns nur für y, x und z ignorieren wir
        var Point(_, y, _) = p;
        System.out.println("Nur y interessiert: " + y);

        // Pattern Matching im Switch
        Object obj = new Point(1,2,3);

        switch(obj) {
            case Point(_, int yVal, _) -> System.out.println("y ist " + yVal);
            default -> System.out.println("Kein Point");
        }
    }
}
