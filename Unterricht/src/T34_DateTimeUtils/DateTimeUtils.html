LocalDateTime

Beschreibung: Stellt Datum und Uhrzeit ohne Zeitzone dar, z. B. “2025-11-10T13:45”.

Erstellen & Nutzen:

  • Statische Methoden:

    • LocalDateTime.now(), LocalDateTime.of(...), LocalDateTime.parse(...) sind statische Methoden zur Erzeugung von Objekten.

LocalDateTime now = LocalDateTime.now();  // aktueller Zeitpunkt
LocalDateTime custom = LocalDateTime.of(2025, 2, 15, 14, 30);  // 15.02.2025, 14:30
LocalDate date = LocalDate.of(2025, 2, 15);
LocalTime time = LocalTime.of(14, 30);
LocalDateTime combined = LocalDateTime.of(date, time); // Kombiniert Datum & Zeit

// Verschieben:
LocalDateTime shifted = custom.plusDays(2).plusHours(3);

// Einzelteile extrahieren:
int year = custom.getYear();
int minute = custom.getMinute();

  • Immutability:

    • LocalDateTime ist immutable. Methoden wie plusDays(), minusHours() erzeugen immer ein neues Objekt, der ursprüngliche Wert bleibt unverändert.

  • Manipulation:

    • Datums- und Zeitänderungen erfolgen durch Methoden, die neue Instanzen mit modifizierten Werten zurückgeben:

    • LocalDateTime dt = LocalDateTime.of(2025, 11, 11, 12, 0); 
LocalDateTime shifted = dt.plusDays(3);

    • Das Originalobjekt dt bleibt unverändert.

Alt vs Neu: Früher wurde meist java.util.Date oder Calendar verwendet – beide sind weniger intuitiv, nicht threadsicher und schwer formatierbar. Die neuen Klassen sind immutable, klar strukturiert und einfacher zu bedienen.

Typische Exceptions:

  • DateTimeException: Tritt auf bei ungültigen Datums-/Zeitwerten, z. B. Monat=13, Stunde=25 oder Datum vor 1 n.Chr.

  • NullPointerException: Wenn z. B. ein null LocalDate oder LocalTime beim Kombinieren übergeben wird.

  • DateTimeParseException: Beim Parsen von Strings mit falschem Format durch LocalDateTime.parse(...).

LocalDate

Beschreibung: Repräsentiert ein Datum ohne Uhrzeit (z. B. “2025-11-11”).

Erstellen & Nutzen:

  • Statische Methoden:

    • Erzeugung über LocalDate.now(), LocalDate.of(...), LocalDate.parse(...).
LocalDate heute = LocalDate.now();
LocalDate geburtstag = LocalDate.of(1990, 10, 1);

boolean gleich = heute.equals(geburtstag);
boolean nachher = heute.isAfter(geburtstag);
boolean vorher = heute.isBefore(geburtstag);

// Hinzufügen/Subtrahieren:
LocalDate zukunft = heute.plusYears(1);
DayOfWeek wochentag = heute.getDayOfWeek();
LocalDate date = LocalDate.of(2025, Month.DECEMBER, 25);
System.out.println(date);  // Ausgabe: 2025-12-25
LocalDate date2 = LocalDate.of(2025, Month.DECEMBER, 32); //Exception

  • Immutable:

    • Änderungen wie plusDays() oder minusMonths() liefern neue Objekte zurück.

  • Manipulation:

    • LocalDate kann durch hinzufügen oder subtrahieren modifiziert werden, z. B.:
LocalDate today = LocalDate.now(); 
LocalDate nextWeek = today.plusWeeks(1);

Alt vs Neu: Mit Date/Calendar musste man viel umständlicher parsen, rechnen und Wochentage als Zahl (0=Sonntag!) herausfinden – jetzt gibt es klare, selbstbeschreibende Methoden.

Typische Exceptions:

  • DateTimeException: Ungültige Werte wie Monat=0 oder Tag=32 führen zu Fehlern.

  • DateTimeParseException: Falsches Format beim Parsen aus String.

  • UnsupportedTemporalTypeException: Beim Versuch, nicht unterstützte Felder (z.B. Stunden bei LocalDate) abzufragen.

LocalTime

Beschreibung: Nur Uhrzeit ohne Datum und Zeitzone, etwa “13:45”.

Erstellen & Nutzen:

LocalTime jetzt = LocalTime.now();
LocalTime festenZeit = LocalTime.of(9, 15);  // 9:15
LocalTime spaeter = festenZeit.plusMinutes(90);  // 10:45

// Vergleich:
boolean vorher = festenZeit.isBefore(spaeter);

Alt vs Neu: Früher wurde meist mit Date gerechnet und mit Formatstrings manipuliert. Jetzt können gezielt Zeitangaben ohne Datum bearbeitet werden.

Typische Exceptions:

  • DateTimeException: Ungültige Uhrzeitwerte, z. B. Minute=61.

  • DateTimeParseException: Falscher Zeitformat-String beim Parsen.

  • UnsupportedTemporalTypeException: Wenn man z.B. Tagesinformationen anfordert.

plusMonths/minusMonths X
plusWeeks/minusWeeks X
plusDays/minusDays X
plusHours/minusHours X
plusMinutes/minusMinutes X
plusSeconds/minusSeconds X
plusNanos/minusNanos X

DateTimeFormatter

Beschreibung: Dient dazu, Datum-/Zeitobjekte zu formatieren und zu parsen (String ↔ Objekt).

  • Statische Fabrikmethoden für Formatierer, z. B.:

    • DateTimeFormatter.ofPattern(...), DateTimeFormatter.ISO_LOCAL_DATE, alle statisch erzeugt.

Beispiele:

DateTimeFormatter fmt = DateTimeFormatter.ofPattern("dd.MM.yyyy HH:mm");
String text = LocalDateTime.now().format(fmt);      // "11.11.2025 09:45"
LocalDateTime ldt = LocalDateTime.parse("15.02.2025 14:30", fmt);

Es gibt vordefinierte Formate (ISO, BASIC), individuelle Muster und die Formatter sind threadsicher (anders als SimpleDateFormat).

ISO_DATE

  • InternationalerStandard für Datum

  • Zeit darf nicht mitgegeben werden

ISO_TIME

  • InternationalerStandard für Zeit

  • Datum darf nicht mitgegeben werden

ISO_DATE_TIME

  • Internationaler Standard für Datum und Zeit

  • Zeit und Datum MÜSSEN mitgegeben werden

Immutable:

  • Instanzen sind unveränderlich und threadsicher.

Verwendung:

  • Formatieren und Parsen von Date-/Time-Objekten mit format() und parse().

Typische Exceptions:

  • IllegalArgumentException: Beim Erstellen eines Formatters mit ungültigem Muster.

  • DateTimeParseException: Wenn das zu parsende Datum/Zeit nicht zum Muster passt.

Period

Beschreibung: Period beschreibt den Unterschied (in Jahren, Monaten, Tagen) zwischen zwei Datumswerten.

  • Statische Erzeugung:

    • Period.between(...) und Period.of(...) sind statische Fabrikmethoden.

  • Immutable:

    • Period-Objekte repräsentieren konstante Zeitspanne (Jahre, Monate, Tage).

  • Manipulation:

    • Um Perioden zu verändern, werden neue Instanzen erzeugt.

Beispiel:

LocalDate geb = LocalDate.of(2000, 3, 1);
LocalDate heute = LocalDate.now();
Period alter = Period.between(geb, heute);
int jahre = alter.getYears();    // Jahre
int monate = alter.getMonths();  // Monate
int tage = alter.getDays();      // Tage

Typisch für Alters-, Vertrags-, oder Jubiläumsberechnung.

​Typische Exceptions:

  • Selten eigene Exceptions, aber NullPointerException bei null-Übergabe.

  • DateTimeException bei ungültiger Perioden-Erstellung.

Duration

Beschreibung: Duration misst exakte Zeitspannen (Stunden, Minuten, Sekunden, Millisekunden), vor allem zwischen Instanzen von LocalDateTime, Instant, LocalTime.

  • Statische Erzeugung:

    • Duration.between(...), Duration.ofHours() sind statische Methoden.

  • Immutable:

    • Dauer ist unveränderlich; Manipulationen liefern neue Instanzen.

Beispiel:

LocalDateTime start = LocalDateTime.of(2025,1,1,0,0);
LocalDateTime ende = LocalDateTime.of(2025,1,2,2,30);
Duration dauer = Duration.between(start, ende);
long stunden = dauer.toHours(); // z.B. 26h

Typisch für Messungen in Maschinen, Stoppuhr, Laufzeit-Auswertungen usw.

Typische Exceptions:

  • ArithmeticException: Bei Berechnung von extrem großen Zeitspannen (Overflow).

  • NullPointerException bei null-Dates.

  • DateTimeException bei ungültigen Eingabewerten.

Period vs Duration

  • Period: Jahre, Monate, Tage (z. B. Vertragsdauer, Alter, Kalenderabstände)

  • Duration: Exakte Zeitdifferenz in Stunden, Minuten, Sekunden (z. B. Maschinenlaufzeit, Zeitmessung, digitale Uhren)

  • Beispiel: Von 1.2.2023 bis 3.3.2025 → Period: 2 Jahre, 1 Monat, 2 Tage. Gleicher Zeitraum in Duration: Z.B. 759 Tage/18216 Stunden.

Vorteile der neuen API gegenüber der alten (Date/Calendar)

  • Immutable: sicher, kein ungewolltes Verändern im Hintergrund

  • Übersichtlich: klare, selbsterklärende Methoden

  • Präzise Typen: LocalDate, LocalTime usw. verhindern „vermischte“ Angaben

  • Bessere Formatierung: DateTimeFormatter ist flexibel und threadsicher

  • Einfacheres Berechnen & Vergleichen: keine Zahlenspiele mehr für Monate, Jahre etc.

  • Klare Trennung: „Datum“, „Uhrzeit“, „Zeitspanne“ als eigene, passende Klasse
