Regular Expressions (Regex) in Java

Was ist Regex?     Eine Regex (Regulärer Ausdruck) ist ein Textmuster, mit dem man Zeichenketten durchsuchen, filtern und validieren kann.

Alltagsbeispiele für Regex-Nutzung

  • Prüfen, ob eine E-Mail-Adresse gültig ist

  • Eine Telefonnummer im Text finden

  • Alle Zahlen aus einem String herausfiltern

  • Prüfen, ob ein Passwort stark genug ist

  • Daten bereinigen, z. B. Leerzeichen oder Sonderzeichen entfernen

Verwendung in Java:      Java bietet seit Anfang an Klassen im Paket java.util.regex mit Pattern und Matcher.

Grundlagen:

Symbol Bedeutung Beispiel Findet
. Beliebiges Zeichen a.b „acb“, „a9b“
* 0 oder mehr Wiederholungen go*d „gd“, „good“, „goooood“
+ 1 oder mehr Wiederholungen ha+ „ha“, „haa“, „haaa“
{n,m} Bereich der Wiederholungen \d{2,4} „12“, „1234“

[] Zeichenklasse [abc] „a“, „b“, „c“
[^abc] Nicht diese Zeichen [^abc] alles außer a,b,c
\d Ziffer (0–9) \d\d „12“, „45“
\w Buchstabe oder Zahl \w+ „abc“, „A9“
\s Leerzeichen \s+ „ “, „\t“
` ` Oder `Katze
() Gruppe (ab)+ „ab“, „abab“

Mit Pattern.compile(regex) kann ein Muster erstellt werden, der später mehrfach verwendet wird.

Mit Matcher können einzelne Treffer durchsucht oder ersetzt werden. import java.util.regex.; public class ExtractNumbers { public static void main(String[] args) { String text = "Artikel 42 kostet 15 Euro und wiegt 2kg"; Pattern pattern = Pattern.compile("\d+"); Matcher matcher = pattern.matcher(text); while (matcher.find()) { System.out.println("Gefundene Zahl: " + matcher.group()); } } } / Ausgabe: Gefundene Zahl: 42 Gefundene Zahl: 15 Gefundene Zahl: 2 */

Generics in Java

Generics ermöglichen es, Klassen, Interfaces und Methoden so zu schreiben, dass sie mit beliebigen Datentypen arbeiten können, ohne dabei die Typ-Sicherheit zu verlieren.

Generics ermöglichen es also, Klassen und Methoden typ-sicher für verschiedene Objekttypen wiederzuverwenden, ohne mehrfachen Code zu schreiben.

👉 Einfach gesagt:

Mit Generics schreibst du eine Klasse oder Methode nur einmal, kannst sie aber mit verschiedenen Datentypen verwenden.

Problem vor Java 5 (ohne Generics):

  • Collections (z. B. List, Map, Set) speicherten nur Object-Typen.

  • Beim Auslesen musste man casten ((String) list.get(0)).

  • Das war fehleranfällig, da Casts erst zur Laufzeit geprüft werden.

Beispiel ohne Generics: List zahlen = new ArrayList(); zahlen.add(10); zahlen.add(20); zahlen.add("30"); // Compiler erlaubt das for (Object o : zahlen) { int i = (int) o; // ClassCastException bei "30" System.out.println(i); }

Kein Compiler-Fehler! Erst zur Laufzeit knallt es.

Beispiel mit Generics: List zahlen = new ArrayList<>(); zahlen.add(10); zahlen.add(20); // zahlen.add("30"); // Compiler-Fehler! for (int i : zahlen) { System.out.println(i); // kein Cast nötig, sicher }

Jetzt prüft der Compiler schon beim Übersetzen, ob die Typen passen. Kein Cast nötig, keine Laufzeitfehler wegen falscher Typen.

Generische Klassen:

Generics führen Typparameter ein — Platzhalter für einen konkreten Datentyp. public class Box { private T item; public void set(T item) { this.item = item; } public T get() { return item; } }

  • T steht für Type Parameter (z. B. String, Integer, Person …)

  • Wenn du Box<String> benutzt, ersetzt der Compiler T durch String //Verwendung Box box = new Box<>(); box.set("Hallo Welt"); String inhalt = box.get(); // Kein Cast nötig!

Generische Methoden haben Typparameter vor dem Rückgabetyp,

z.B.: public static void printArray(T[] array) { ... } //Verwendung String[] namen = {"Anna", "Ben", "Clara"}; Integer[] zahlen = {1, 2, 3}; printArray(namen); printArray(zahlen);

Syntax Bedeutung
T Beliebiger Typ (Type Parameter)
<T> Definition eines generischen Typs
<?> Unbekannter Typ (Wildcard)
<T extends Number> Eingeschränkter Typ (Bounded Type)
<K, V> Mehrere generische Typen
