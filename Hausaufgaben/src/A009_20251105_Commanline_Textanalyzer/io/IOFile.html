Skript: Grundlagen von FileReader, BufferedReader und Writer in Java

Lernziele

Nach diesem Kapitel kannst du:

  • Dateien in Java lesen und schreiben

  • Den Unterschied zwischen FileReader und BufferedReader erklären

  • Wissen, wann und warum man Puffer (Buffer) einsetzt

  • Das Dekorator-Pattern in diesem Zusammenhang verstehen

  • Eigene Programme schreiben, die mit Textdateien arbeiten

Einführung: Warum mit Dateien arbeiten?

Bisher haben wir Daten vielleicht nur über die Konsole ein- und ausgegeben.
In der Praxis müssen Programme aber oft dauerhaft Daten speichern – z. B.:

  • Konfigurationsdateien

  • Log-Dateien

  • Textdateien mit Benutzerdaten

Dafür verwendet man in Java die IO-Klassen aus dem Paket java.io.

FileReader

Ein FileReader liest Zeichen (char) aus einer Datei. Er ist also für Textdateien gedacht, nicht für Binärdaten wie Bilder.

Beispiel: 

import java.io.FileReader;
import java.io.IOException;

public class SimpleFileReader {
    public static void main(String[] args) {
        try {
            FileReader reader = new FileReader("beispiel.txt");
            int zeichen;
            while ((zeichen = reader.read()) != -1) {
                System.out.print((char) zeichen);
            }
            reader.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

Erklärung:

  • read() liest ein Zeichen (als int) und gibt -1 zurück, wenn das Ende erreicht ist.

  • Das (char) wandelt den int-Wert wieder in ein Zeichen um.

  • Mit close() wird die Datei wieder freigegeben.

Nachteil: Das Lesen Zeichen für Zeichen ist ineffizient!

BufferedReader

Ein BufferedReader arbeitet mit einem Zwischenspeicher (Buffer). Er liest intern größere Datenblöcke auf einmal und macht das Lesen damit viel schneller.

import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class BufferedReaderExample {
    public static void main(String[] args) {
        try {
            BufferedReader br = new BufferedReader(new FileReader("beispiel.txt"));
            String zeile;
            while ((zeile = br.readLine()) != null) {
                System.out.println(zeile);
            }
            br.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

Erklärung:

  • readLine() liest eine ganze Zeile bis zum Zeilenumbruch.

  • Praktischer und schneller als Zeichen für Zeichen.

  • Intern nutzt der BufferedReader den FileReader – das ist ein Beispiel für das Dekorator-Pattern!

Exkurs: Das Dekorator-Pattern

Das Dekorator-Pattern (Dekorierer-Muster) erlaubt es, Objekte dynamisch mit zusätzlicher Funktionalität zu „verpacken“.

Unser Beispiel: 

BufferedReader br = new BufferedReader(new FileReader("beispiel.txt"));

Das Decorator Pattern erlaubt es, Objekte zur Laufzeit zu erweitern, indem sie in andere Objekte „eingewickelt“ (wrapped) werden.

Prinzip:

Reader reader = new BufferedReader( new FileReader("text.txt"));

FileReader ist die Basiskomponente, die tatsächlich aus der Datei liest. BufferedReader ist der Dekorator, der zusätzliche Funktionalität (Pufferung, readLine()) hinzufügt.

Beide implementieren das gleiche Interface (Reader), daher können sie austauschbar verwendet werden.

+--------------------+ | Reader | ← Abstrakte Komponente +--------------------+ ▲ │ +----------------------+ | FileReader | ← Konkrete Komponente +----------------------+ ▲ │ +--------------------------+ | BufferedReader | ← Dekorator, erweitert Reader +--------------------------+

Der BufferedReader ändert nicht den Typ, sondern erweitert das Verhalten – das ist typisch für das Decorator Design Pattern.

Dateien schreiben mit FileWriter und BufferedWriter

Wie beim Lesen gibt es auch fürs Schreiben:

  • FileWriter → schreibt einzelne Zeichen

  • BufferedWriter → schreibt zeilenweise und schneller

import java.io.FileWriter;
import java.io.IOException;

public class SimpleFileWriter {
    public static void main(String[] args) {
        try {
            FileWriter writer = new FileWriter("ausgabe.txt");
            writer.write("Hallo Welt!\n");
            writer.write("Das ist eine zweite Zeile.");
            writer.close();
            System.out.println("Datei erfolgreich geschrieben!");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

import java.io.BufferedWriter;
import java.io.FileWriter;
import java.io.IOException;

public class BufferedWriterExample {
    public static void main(String[] args) {
        try {
            BufferedWriter bw = new BufferedWriter(new FileWriter("ausgabe.txt"));
            bw.write("Hallo mit Buffer!");
            bw.newLine(); // fügt eine neue Zeile hinzu
            bw.write("Das Schreiben ist jetzt schneller!");
            bw.close();
            System.out.println("Datei erfolgreich geschrieben!");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

Erklärung:

  • newLine() fügt betriebssystemunabhängig eine neue Zeile ein.

  • Durch den internen Puffer werden Daten erst beim Schließen oder wenn der Buffer voll ist, wirklich geschrieben.

Zusammenfassung

Klasse Zweck Bemerkung
FileReader Liest Zeichen aus Datei Basis-Klasse
BufferedReader Liest zeilenweise, gepuffert Dekoriert FileReader
FileWriter Schreibt Zeichen in Datei Basis-Klasse
BufferedWriter Schreibt gepuffert, schneller Dekoriert FileWriter

Übungsaufgaben

Aufgabe 1 – Datei lesen

Schreibe ein Programm, das die Datei text.txt einliest und alle Zeilen nummeriert auf der Konsole ausgibt.
(Beispielausgabe: 1: Erste Zeile, 2: Zweite Zeile, …)

Aufgabe 2 – Datei schreiben

Erstelle ein Programm, das den Benutzer nach seinem Namen fragt und ihn in die Datei namen.txt schreibt.
Jeder Aufruf soll den Namen anhängen, nicht überschreiben.

💡 Tipp: new FileWriter("namen.txt", true) aktiviert den Anhangmodus.

Aufgabe 3 – Datei kopieren

Schreibe ein Programm, das den Inhalt einer Datei (quelle.txt) in eine neue Datei (ziel.txt) kopiert – Zeile für Zeile mit BufferedReader und BufferedWriter.

Bonus: Alternativen (ab Java 7)

Mit den Klassen aus java.nio.file (z. B. Files.readAllLines() oder Files.write()) geht vieles einfacher – aber es lohnt sich, die Grundlagen mit FileReader & Co. zu verstehen, um zu wissen, wie IO in Java funktioniert.
